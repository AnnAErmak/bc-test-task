При выполнении данного кода консоль выведет четыре раза Bad: undefined.
Это происходит из-за того что переменная i всплывает в глобальную область видимости,
так как объявлена через ключевое слово var. На каждой итерации цикла for она меняет
свое значение и на момент выполнения кода становится равна 4. Затем мы пытаемся сравнить
arr[4] > 13, arr[4] вернет undefined, т.е undefined > 13 это false, получаем Bad: undefined

Для того, чтобы код работал как в тз, можно сделать следующее:

1. Использование объявление переменной i через ключевое слово let. Тогда переменная
будет иметь блочную область видимости.

2. Передать еще один параметр в setTimeout
const arr = [10, 12, 15, 21]

for (var i = 0; i< arr.length; i++){
    setTimeout(function (i){
        console.log(arr[i] > 13 ? `Good: ${arr[i]}` : `Bad: ${arr[i]}`)
    }, 1000, i)
}

3. Использовать немедленно вызываемую функцию, чтобы сохранить текущее занечение i
const arr = [10, 12, 15, 21]

for (var i = 0; i< arr.length; i++){
    (function (i){
        setTimeout(function (){
            console.log(arr[i] > 13 ? `Good: ${arr[i]}` : `Bad: ${arr[i]}`)
        }, 1000)
    })(i)
}
